# 11 gogo.gs 油價整合可行性研究（定案）

> 2026-07-07 實查。結論：**不整合，油價／旅程成本走手動輸入。** 這是定案，不是暫定；除非 gogo.gs 重新推出官方 API，不要再開這個題目。

## 四關判準逐條回答（03 檔 §11）

| 關卡 | 結果 | 證據 |
|---|---|---|
| 1. 官方 API 或資料下載 | ❌ 不過 | 官方 API 頁 http://api.gogo.gs/ 本文明載：「ガソリン価格情報 WEBサービスAPIはサポートを終了しています。」（曾存在，已終止支援；2026-07-07 curl 實讀） |
| 2. ToS/robots 允許程式化取用 | ⚠️ 未逐條讀 | robots.txt 對非瀏覽器 fetcher 回 403、帶瀏覽器 UA 才回 200 → 網站主動擋程式化存取，訊號負面。第 1 關已不過，不需再深查 |
| 3. 不需 JS／人機驗證 | 未測 | 同上，不需再測 |
| 4. 失敗可偵測 | 不適用 | 無合法來源可整合 |

## 定案方案：手動輸入（已與現行 Sheet 邏輯一致）

成本 Sheet「規劃成本暫存區」本來就有 `油費（日圓）fuel_jpy` 手填欄（見 `10-cost-sheet-mapping.md` §2.1，F 欄），老闆每趟行程填一次。遷移到系統時照抄這個模式：

1. 旅程成本表單保留 `fuel_jpy`（油費）手動輸入欄；UI 附 gogo.gs 連結（https://gogo.gs/ ，一般瀏覽器可用）供老闆查價後手填。
2. 交通分攤沿用 Sheet 現行公式口徑（ETC＋電車＋油費＋停車 → 除以預估件數；原文公式在 10 檔 §2.1）。
3. 欄位帶「填寫日期」；不做過期告警以外的任何自動化。
4. 禁止事項不變：不寫 gogo.gs 爬蟲、不繞 UA、不解析其 HTML（AGENTS.md 最小規則 6）。
