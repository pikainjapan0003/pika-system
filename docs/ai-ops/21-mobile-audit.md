- 稽核方式：依 `14_UI_UX_CONSTITUTION.md` §3 靜態檢查目前 React 結構與 Tailwind 樣式；本報告不修改程式。

## 1. 客人下單頁

### 通過

- 根容器為手機單欄，寬度上限 480px：`artifacts/shop-app/src/pages/PublicOrder.tsx:478`。
- 加入購物車及購物車入口皆為 44px：`PublicOrder.tsx:594`、`:604`。
- 最終送出按鈕為滿版 48px，且黏在拇指容易觸及的下方：`PublicOrder.tsx:991`。
- 金額摘要採直式卡片，沒有寬表格或需要橫向捲動的客人介面。

### 缺陷

1. 規格選項僅 `px-4 py-2`，沒有 44px 最小高度：`PublicOrder.tsx:553`。
2. 數量加減按鈕為 40×40px：`PublicOrder.tsx:573`、`:581`。
3. 兩個門市選擇按鈕為 40px 高：`PublicOrder.tsx:785`、`:835`。

建議後續只做 UI 小包：上述可點擊控制統一 `min-h-11`，不改下單邏輯。

## 2. 客人查單頁

### 通過

- 付款末五碼輸入與送出按鈕均為 44px：`artifacts/shop-app/src/pages/TrackOrder.tsx:442`、`:448`。
- 複製連結與返回按鈕均為滿版 44px：`TrackOrder.tsx:463`、`:471`。
- 狀態、付款與門市資訊均採單欄卡片，沒有後台表格感。
- 「待確認」等狀態以文字呈現，未把未知金額默示為 0。

### 缺陷

1. 錯誤狀態的「返回」是文字型按鈕，未設 44px 觸控高度：`TrackOrder.tsx:194-199`。
2. 頁首返回按鈕同樣未設 44px 觸控區：`TrackOrder.tsx:214-218`。

## 3. 老闆訂單列表

### 通過

- 根容器以 480px 為上限：`artifacts/shop-app/src/pages/Orders.tsx:667`。
- 新增訂單、包貨清單、搜尋與主要篩選控制為至少 44px：`Orders.tsx:671-745`。
- 狀態篩選列可橫向捲動，不會撐破 375px：`Orders.tsx:745`。
- 六張摘要卡在手機為兩欄：`Orders.tsx:769`。
- 底部批次工具列固定於拇指區，主要控制均至少 44px：`Orders.tsx:1555-1651`。

### 缺陷

訂單卡片內仍有多組低於 44px 的高頻按鈕：

- 選擇／變更門市 36px：`Orders.tsx:1269`、`:1314`。
- 訊息範本 40px：`Orders.tsx:1407`。
- 編輯、複製連結、列印收據均 36px：`Orders.tsx:1426`、`:1439`、`:1448`。
- 狀態與危險操作均 36px：`Orders.tsx:1487`、`:1515`、`:1524`。

建議後續將卡片內可點擊動作統一為 `min-h-11`，危險操作保留二段確認。

## 4. 包貨清單

### 通過

- 彈窗為 88dvh、最大 480px，內文可獨立縱向捲動：`artifacts/shop-app/src/pages/PickingListDialog.tsx:24`、`:67`。
- 商品、規格、SKU 與數量採卡片排列，375px 不需寬表格。

### 缺陷

1. 列印按鈕僅 28px：`PickingListDialog.tsx:38`。
2. 關閉按鈕僅 36×36px：`PickingListDialog.tsx:46`。
3. 目前是「可列印的彙總清單」，沒有每個品項的勾選控制，也沒有未包／已包的視覺狀態；因此不符合憲法所要求的「勾已包」最短包貨流程。這是功能 UX 缺口，不宜在本報告包順手修改。

## 建議施工順序

1. **P2：包貨勾選流程**——先拍板勾選狀態只存在本機 UI，還是要永久存 DB；未拍板前不可實作。
2. **P3：訂單卡片觸控區**——所有 36–40px 動作改為至少 44px。
3. **P3：客人頁次要控制觸控區**——規格、數量、選店與返回按鈕改為至少 44px。
