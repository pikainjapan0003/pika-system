# Order Step 5F — 撿貨單 / 出貨單 / 匯出 / 列印 規格文件

> 版本：Step 5F-A（規格文件）｜分支：`order-step5f-picking-shipping-export-spec`
> 本文件為規格與任務切分，不含實作。
> 文件語言：繁體中文。

---

## 1. Step 5F 目標

Step 5F 是 Step 5（付款 / 物流欄位）完成後的下一個功能階段。目標是讓店家可以：

| 功能 | 說明 |
|------|------|
| **撿貨單** | 依商品彙總所有待出貨訂單的品項與數量，供包貨使用 |
| **出貨單** | 依訂單列出收件人資訊、商品、付款 / 出貨狀態，供出貨核對 |
| **CSV 匯出** | 將篩選後的訂單匯出為 CSV，供 Excel / Google Sheets 後續處理 |
| **列印格式** | 提供適合紙本列印的頁面（A4，無多餘色彩），讓老闆可直接列印夾附包裹 |
| **PDF（後續）** | MVP 先不做，後續若有需求再評估伺服器端 PDF 產生 |

---

## 2. 使用者情境

### 情境 A：批次整理待出貨訂單

老闆在 `/orders` 後台篩選出「出貨狀態 = pending / processing」的訂單，想一次看到所有需要出貨的品項，確認庫存是否足夠。

**需求**：從篩選結果產生撿貨單，以商品為單位彙總數量。

### 情境 B：依商品彙總撿貨

老闆準備包裹時，需要知道「這批訂單一共要準備幾個商品 A、幾個商品 B」，而不是逐筆訂單查。

**需求**：撿貨單依商品彙總，顯示每個商品的總數量與訂單列表。

### 情境 C：依訂單列出收件 / 取貨資訊

老闆需要在包好的包裹上貼上出貨單，或交給物流業者時核對收件人資訊。

**需求**：出貨單依訂單為單位，列出收件人、商品、物流資訊。

### 情境 D：匯出 CSV 給 Excel / Google Sheets

老闆需要將訂單資料帶出系統，做進一步整理、對帳、或與配送廠商溝通。

**需求**：CSV 匯出，欄位齊全，編碼正確（UTF-8 BOM 以相容 Excel），日期與金額格式固定。

### 情境 E：列印紙本給包貨使用

老闆或助手包裝時，手邊有一份紙本清單，可以逐項打勾確認。

**需求**：適合 A4 列印的頁面，無多餘背景色，狀態 badge 轉為文字顯示。

## 5. 撿貨單資料欄位建議

撿貨單以**商品為單位**彙總，跨多筆訂單合計數量。

| 欄位 | 說明 | 必要 |
|------|------|------|
| 商品名稱 | `product.name` | ✅ |
| 商品編號 / SKU | `product.sku` | ✅ |
| 規格 / 品項備註 | `orderItem.note`（若有） | 選填 |
| 數量合計 | 所有訂單同商品的 `quantity` 加總 | ✅ |
| 溫層 | `product.storageTemp`（常溫 / 冷藏 / 冷凍） | ✅ |
| 保存期限 | `product.shelfLife` | 選填 |
| 單位重量 | `product.weightKg` | 選填 |
| 對應訂單編號 | 包含此商品的所有 `order.id` 或 `order.orderNumber` | ✅ |
| 備註 | 各訂單的商品備註（若不同則列出） | 選填 |

> **待確認**：撿貨單是否包含已取消（cancelled）的訂單？建議預設**不包含**，但需讓店家可選擇。

## 6. 出貨單資料欄位建議

出貨單以**訂單為單位**，每筆訂單一份。

### 訂單基本資訊

| 欄位 | 來源 | 顯示 |
|------|------|------|
| 訂單編號 | `order.id` / `order.orderNumber` | ✅ 必要 |
| 下單時間 | `order.createdAt` | ✅ |
| 付款狀態 | `order.paymentStatus` | ✅ |
| 付款方式 | `order.paymentMethod` | ✅ |
| 出貨狀態 | `order.shippingStatus` | ✅ |
| 物流方式 | `order.shippingMethod` | ✅ |
| 物流追蹤碼 | `order.trackingCode` | 若有則顯示 |

### 買家 / 收件人資訊

| 欄位 | 來源 | 顯示 | 個資注意 |
|------|------|------|----------|
| 買家姓名 | `order.customerName` | ✅ | 內部文件可顯示 |
| 買家電話 | `order.customerPhone` | ✅ | 內部文件可顯示 |
| 收件人姓名 | `order.recipientName` | ✅ | 內部文件可顯示 |
| 收件人電話 | `order.recipientPhone` | ✅ | 內部文件可顯示，**完整顯示**（非遮罩） |
| 收件地址 | `order.recipientAddress` | ✅ | 內部文件可顯示 |
| 超商店號 | `order.cvsStoreId` | 超商取貨時顯示 |  |
| 超商店名 | `order.cvsStoreName` | 超商取貨時顯示 |  |

> **說明**：出貨單屬於**店家內部使用文件**，個資欄位（電話、地址）應完整顯示以利配送。
> 與公開查詢頁規則不同，公開查詢頁須遮罩個資。

### 商品明細

| 欄位 | 來源 | 顯示 |
|------|------|------|
| 商品名稱 | `product.name` | ✅ |
| 規格 / 備註 | `orderItem.note` | 若有 |
| 數量 | `orderItem.quantity` | ✅ |
| 單價 | `orderItem.unitPrice` | ✅ |
| 小計 | `orderItem.subtotal` | ✅ |

### 金額資訊

| 欄位 | 來源 | 顯示 |
|------|------|------|
| 商品小計 | `order.subtotal` | ✅ |
| 運費 | `order.shippingFee` | ✅ |
| 訂單總額 | `order.totalAmount` | ✅ |
| 實際付款金額 | `order.paidAmount` | ✅ |
| 待收金額 | `order.amountDue` | ✅ |

### 備註欄位

| 欄位 | 是否顯示在出貨單 | 說明 |
|------|-----------------|------|
| `order.internalNote` | **待確認** | 屬於店家內部備註，視需求決定 |
| `order.paymentNote` | **待確認** | 付款備註，可能含帳戶資訊，需謹慎 |
| `order.trackingCode` | ✅ 顯示 | 物流追蹤碼 |

> **待確認**：`internalNote` 是否列印於出貨單。建議提供開關讓店家自行決定。

## 7. 匯出格式（CSV）

### 欄位定義

建議 CSV 欄位順序：

```
訂單編號, 下單時間, 買家姓名, 買家電話, 收件人姓名, 收件人電話, 收件地址,
超商店號, 超商店名, 物流方式, 物流追蹤碼, 出貨狀態, 付款方式, 付款狀態,
商品名稱, 規格, 數量, 單價, 小計, 運費, 訂單總額, 實際付款, 待收金額,
下單備註, 內部備註（選填）
```

### 編碼

- **UTF-8 BOM**（`﻿` 開頭）
- 原因：Windows Excel 預設以 BOM 判斷編碼，無 BOM 會導致中文亂碼

### 金額格式

- 整數，單位：元（新台幣）
- 不含貨幣符號、不含千分位逗號（避免 CSV 解析錯誤）
- 範例：`250`（不是 `NT$250` 或 `250,000`）

### 日期格式

- ISO 8601：`2026-06-05T14:30:00+08:00`
- 或台灣慣用格式：`2026-06-05 14:30`（依產品決策）

### CSV Escape 規則

- 欄位內含逗號 → 整個欄位用雙引號包住：`"台北市大安區，某某路"`
- 欄位內含雙引號 → 雙引號重複：`"他說""好"""`
- 欄位內含換行 → 整個欄位用雙引號包住（或移除換行）

### 個資注意

- CSV 屬於**店家內部使用**，可包含完整個資（電話、地址）
- 不可設計成公開下載端點
- API 端點需要認證（Bearer token），未登入回 401

## 8. 列印格式

### 頁面規格

| 項目 | 建議 |
|------|------|
| 頁面尺寸 | A4（210 × 297 mm） |
| 邊距 | 10–15 mm |
| 字體大小 | 本文 10–11pt，標題 12–14pt |
| 顏色 | 黑白為主，無背景色，badge 轉為文字 |
| 頁碼 | 多頁時顯示頁碼 |

### 技術實作建議

- 使用 CSS `@media print` 控制列印樣式
- 不使用深色背景 / 彩色 badge，改為文字括號標示（如：`[待付款]`、`[已出貨]`）
- 避免使用絕對定位導致列印時跑版
- 每筆出貨單可加 `page-break-after: always` 讓每張訂單從新頁開始

### 預覽

- 桌機：提供列印預覽按鈕（`window.print()`）
- 手機：列印預覽頁需可捲動，字體不可過小

## 9. 權限與安全

| 規則 | 說明 |
|------|------|
| 認證必要 | 所有撿貨單 / 出貨單 / CSV API 需要 Bearer token 認證 |
| 店家隔離 | 只能匯出自己店家的訂單，不可存取其他店家資料 |
| `publicToken` ≠ `trackingCode` | `publicToken` 是買家查詢訂單的 token，不是物流追蹤碼 |
| `internalNote` | 不可出現在任何公開 API 或公開查詢頁，出貨單（內部）是否顯示需明確標註 |
| `paidAmount` | 出貨單（內部）可顯示，公開查詢頁不可顯示 |
| CSV 下載端點 | 不可無認證存取，不可設計為公開 URL |

### 列印頁測試

- [ ] 桌機列印預覽正常
- [ ] A4 頁面不爆版
- [ ] badge 轉為文字顯示
- [ ] 多訂單時每張從新頁開始

### 權限測試

- [ ] 未登入無法下載 CSV
- [ ] A 店家無法匯出 B 店家訂單

## 12. MVP 決策 / 待確認問題

### 12-A. 已確認 MVP 決策（Step 5F-B 可依此施工）

| # | 問題 | MVP 決策 | 原因 |
|---|------|----------|------|
| 1 | 撿貨單是否包含「已取消」的訂單？ | ❌ **不包含** cancelled | 已取消訂單不應進入撿貨流程，避免誤出貨 |
| 2 | 出貨單是否包含「未付款」的訂單？ | ✅ **可以包含**，但必須明確顯示付款狀態 | 部分店家允許貨到付款 / 面交 / 先出後收 |
| 3 | `internalNote` 是否列印在出貨單？ | ❌ **預設不列印**，後續可加選項 | 內部備註可能含敏感資訊，不應預設出現在紙本 |
| 4 | `paymentNote` 是否列印在出貨單？ | ❌ **預設不顯示** | 可能含帳戶資訊，需謹慎 |
| 5 | 收件人電話在出貨單是否完整顯示？ | ✅ **後台出貨單 / CSV 完整顯示**；公開查詢頁仍遮罩 | 出貨需電話聯絡，但公開頁須保護個資 |
| 6 | CSV 是否可包含個資（電話、地址）？ | ✅ **可以**，文件需標註「請勿公開分享 CSV」 | CSV 為登入店家後台的內部匯出，受認證保護 |
| 7 | PDF 功能是否排入 Step 5F？ | ❌ **延後**，先做 browser print / CSV | 伺服器端 PDF 複雜度高，MVP 不需要 |
| 8 | CSV 多品項訂單如何處理？ | ✅ **一訂單一列**，商品明細合併為文字 | 最直覺，後續可加一品項一列模式 |
| 9 | 撿貨單資料來源？ | ✅ **優先支援已選訂單**，後續再支援篩選結果 | 已選訂單最直覺，避免誤匯出全部訂單 |
| 10 | 出貨單資料來源？ | ✅ **使用已選訂單** | 出貨單對應本次要處理的批次 |
