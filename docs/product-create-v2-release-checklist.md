# Product Create V2 — Release Checklist

> 版本：Step 9C｜用途：每次改動商品相關功能後的人工 QA 驗收。
> 直接勾選後截圖或貼入 PR 描述。

---

## A. 商品新增 QA

- [ ] 商品名稱可填
- [ ] 價格可填
- [ ] 庫存可填，或留空代表不限
- [ ] 商品編號（SKU）可填
- [ ] 主分類下拉可選
- [ ] 溫層可選：常溫 / 冷藏 / 冷凍
- [ ] 保存期限可填
- [ ] 重量（g）可填
- [ ] 收單截止時間可設定
- [ ] 商品描述可填
- [ ] 內部備註可填
- [ ] 圖片可上傳，或填入圖片網址
- [ ] 按「建立」後不出現錯誤訊息
- [ ] 建立後商品出現在商品列表

## G. API / Typecheck QA

- [ ] `shop-app` typecheck 通過（`pnpm --filter shop-app typecheck` 或對應指令）
- [ ] `api-server` typecheck 通過

- [ ] API `/healthz` 正常回應

- [ ] public API 不回傳 `internalNote`
- [ ] public API 不回傳後台敏感資訊
- [ ] `storageTemp` / `shelfLife` / `weightKg` 是否公開已標註為產品決策（見下方備注）

> **產品決策備注（storageTemp / shelfLife / weightKg）**

> 本 checklist 不預設答案，請在每次 release 前確認並勾選對應決策。

## I. Release 判斷規則

| 狀態                 | 定義                                                               |
| -------------------- | ------------------------------------------------------------------ |
| **READY**            | 所有 critical 項目通過，沒有 blocking bug。                        |
| **READY WITH NOTES** | 可出貨 / 試跑，但仍有非阻斷性待確認事項。                          |
| **NEEDS WORK**       | 功能大致可用，但有 UI / 文案 / 小流程問題需修。                    |
| **NOT READY**        | 新增商品、編輯商品、下單、截止阻擋、資料安全任一 critical 項失敗。 |
| **BLOCKED**          | 缺環境、缺登入、缺測試資料，無法判斷。                             |

> **Critical 項目定義**（以下任一失敗 → NOT READY）：
>
> - 無法新增商品
> - 無法編輯商品
> - 已截止商品仍可送單（前端或後端）
> - `internalNote` 出現在公開頁
> - secrets / credentials 出現在任何頁面
