## 三、成本這件事，分成兩條線（你 2026-07-07 拍板的）

1. **交通成本分攤線（現在做這條）**：用你的 Google Sheet「規劃成本暫存區」的資料——每趟的 ETC、電車、油費、停車、紙箱、運費、1.5% 手續費，除以預估件數，算出**每件商品該分攤多少基礎交通成本**。只用這個分頁的資料。
2. **整體成本計算線（凍結，先不做）**：營收、毛利、優惠、固定/變異費用分類等。等交通線做完、你想開了再開。

補充定案：**油價不自動抓**（gogo.gs 官方 API 已停止服務），你去 gogo.gs 查完手動填，系統附連結。

### 第 1 步：交通成本分攤模組（AI 做，約 1–2 個工作天的 AI 任務量）

- 系統裡新增「行程（trip）」：一趟北海道 = 一筆，填 ETC／電車／油費／停車／紙箱／運費／預估件數／匯率。
- 系統自動算「每件基礎交通成本」，公式照 Sheet（已抄錄原文），用 Sheet 真實 3 筆資料當測試對照，數字要一模一樣才算過。
- 每個數字都可以人工改（臨時改價、估錯重量都是常態）。
- **驗收方式（你親自做 5 分鐘）**：拿一趟舊行程，把 Sheet 的數字填進系統，系統算出來的單件成本要和 Sheet 一樣。

### 第 2 步：分攤接到商品上

- 上架商品時選「這件是哪一趟帶回來的」，商品自動背上該趟的單件交通成本。
- 驗收：隨便抽 3 件商品，成本 = 日圓成本×匯率 ＋ 該趟分攤，手算對照一致。

### 第 3 步（之後再說，需要你點頭才開）

- 單件毛利顯示（售價 − 成本）→ 每張訂單毛利 → 每趟總毛利。
- 整體成本計算線解凍（回答凍結的 5 個問題後才開工）。
- 貨態自動同步收尾、到貨通知（LINE/email）。
